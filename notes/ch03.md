This chapter explores heaps and red black trees with persistence.

## OCaml translation

- ocaml uses dune as a build system; I set it up to enable importing the ORDERED sig instead of copying it
- I translated the tuples into curried params again; felt right at the time
- `let name = function` is syntax for single (implicit) arg pattern matching
- `match arg1, arg2, ... with` is syntax for pattern matching. [let rec](https://ocaml.org/manual/5.0/expr.html#sss:expr-localdef) puts the name into scope during definition so it can be self-referencing. it can also be used to define mutually recursive functions with `and`. they can be used together to define a recursive function with pattern matching dispatch
- [=](https://ocaml.org/manual/5.0/expr.html#ss:expr-operators) is structural equality, so that's right for immutability tests
- the book's HEAP definition has a comment about the exception; I defined it inline in the LeftistHeap functor for consistency with the book

## Observations

### Leftist heaps

- rank + makeT enforce the O(log n) invariant for LeftistHeap (left >= right) ranks. in the book they are defined separately; I added them in the functor definition which is hidden behind the HEAP type narrowing

### Binomial heaps

- binomial heaps and the FFT both exploit binary decomposition on the structure of 2^k. both of them collapse a naive O(n) or O(n^2) complexity into a log factor per element, but the heap uses it for structure where the FFT uses it for computation. the FFT does this through divide and conquer on DFTs. I find this shared algebraic structure fascinating and it brings me joy
- link enforces heap order and structure by making the heap with a larger root the left most child of the heap with a smaller root. this maps onto the visualization of ranks in the book. noting that the link order would be reversed for a max heap
- the recursive case of merge restructures the tree and generates the structure. the else (3rd) branch links the trees of equal rank which is the binary carry operation mentioned in the book
- the exercises for this chapter are interesting; 3.7 to optimize findMin from O(log n) to O(1) looks fun, but I plan to complete the tour in ocaml and rust before doing these optimizations. optimizing the ocaml early may make it more difficult to look back and understand for rust. the same thinking applies to not storing explicit ranks for 3.6

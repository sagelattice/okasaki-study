# Chapter 3 - Some Familiar Data Structures in a Functional Setting

This chapter explores heaps and red black trees with persistence.

## OCaml translation

- ocaml uses dune as a build system; I set it up to enable importing the ORDERED sig instead of copying it
- I translated the tuples into curried params again; felt right at the time
- `let name = function` is syntax for single (implicit) arg pattern matching
- `match arg1, arg2, ... with` is syntax for pattern matching. [let rec](https://ocaml.org/manual/5.0/expr.html#sss:expr-localdef) puts the name into scope during definition so it can be self-referencing. it can also be used to define mutually recursive functions with `and`. they can be used together to define a recursive function with pattern matching dispatch
- [=](https://ocaml.org/manual/5.0/expr.html#ss:expr-operators) is structural equality, so that's right for immutability tests
- the book's HEAP definition has a comment about the exception; I defined it inline in the LeftistHeap functor for consistency with the book

## Observations

- rank + makeT enforce the O(log n) invariant for LeftistHeap (left >= right) ranks. in the book they are defined separately; I added them in the functor definition which is hidden behind the HEAP type narrowing
- binomial heaps and the FFT both exploit sparse binary decomposition on the structure of 2^k. both of them collapse a renaive O(n) or O(n^2) structure into a log factor through binary halving. I find this shared algebraic structure fascinating and it brings me joy

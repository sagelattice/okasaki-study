# Chapter 6 - Amortization and Persistence via Lazy Evaluation

## OCaml translation

- I had claude refactor collection signatures into sigs.ml, which follows a pattern from the std lib and Jane Street
- This led me to realize that the STREAM sig is going to be heavily reused, so I moved that too. I then made `BankersQueue` a functor over STREAM, which is a departure from the book. I instantiate it in the test and reuse the queue test functor, which is an elegant outcome
- [`lazy pattern`](https://ocaml.org/manual/5.4/patterns.html#sss%3Apat-lazy) forces the suspended computations in the pattern being matched; this is important when doing amortized analysis on lazy data structures, because that syntax would incur any accumulated amortized costs of suspension

## LazyBinomialHeap

- the `lazy pattern` idea above applies here; I misunderstood the syntax when translating from the book. the SML/NJ extension used by the book's code allows for `fun lazy ...` which suspends the application of the function. the pattern matching syntax + strict evaluation meant that the previous translation I wrote did not have the same amortized boundaries as the book. I fixed this by removing the eager forcing and moving the force within the suspensions
- I kept that syntax for `is_empty` because it needs to force the suspension to produce a value
- `find_min` and `delete_min` force $O(\log\ n)$ amortized costs, but `delete_min` defers re-merging the heap children

# Chapter 2 - Persistence

This chapter introduces the concept of persistence and demonstrates it on two simple data structures; lists and binary search trees. I translated the book's code from SML to OCaml.

## OCaml translation

- Claude assisted with OCaml syntax errors and idiomatic review; not logic, structure, or design decisions.
- SML - tuples; OCaml - curried params for function inputs
- OCaml functors for ops on signature vs bare functions in the book
- OCaml functors for light test assertions; I don't know idomatic OCaml testing so that can come later
- OCaml failwith for invalid input states; good enough for a study
- OCaml uses let rec vs fun for matching params

## Observations

- Tests aren't exhaustive; qcheck property testing would be more robust
- It's funny how close cons is to lisp every time I see it
- Some members of ORDERED aren't used (eq, leq), I'm assuming the book introduces it in this chapter and reuses it later

## Persistence mechanism

- List: structural sharing via cons
- BST: copy path on insert so the new path is allocated but shares with the existing structure

Structural sharing is transparent to callers and functors act as the interface.
